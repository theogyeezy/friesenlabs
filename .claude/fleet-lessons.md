# Fleet lessons — friesenlabs repo

Repo-specific lessons for the /fleet skill (conventions, territory couplings, the verify command
that actually works here). One dated line each. Committed so every lane learns from every lane.

2026-06-11 · verify command: `pytest -q` (full) or targeted `pytest <path> -x`; web lane uses `cd web && npm run typecheck && npm run build` + Playwright. CI gate = python (real Postgres+pgvector isolation) · terraform fmt/validate · web · smoke.
2026-06-11 · DB stores open lazy pools now (minconn=1); CI Postgres has limited connection slots — integration tests that each spin a store can still exhaust them under heavy fan-out. Keep DB-touching tasks per-wave modest.
2026-06-11 · `shared/` is BOTH a repo package AND the prefix used in decision-brief `## Sources` citations (provenance to the claude-api skill). Backticked `shared/*.md` mentions in docs/decisions are citations, not repo paths — do NOT "fix" them.
2026-06-11 · territories that conflict at merge: db/schema.sql (EOF-append only, keep all hunks), api/asgi.py + api/app.py (route includes — keep all), db/roles.sql (grants — keep all). RLS-EXEMPT tables (leads/support_requests/workspace_keys) need explicit roles.sql grants — the tenant-table gate won't catch a missing one.
2026-06-11 · customer-readiness modules mostly already have INTEGRATION tests; the genuine unit-coverage gaps were the integration-only ones (billing_routes, support_routes, limits, leads). Scout existing tests/ first and decompose only real gaps — don't re-cover what's covered.
2026-06-11 · CHANGING a shared store method's SQL/signature (api/pg_clients.py) or a route's verb surface ripples into tests OUTSIDE the task territory: `insert_deal` adding a column broke `test_pg_crm_writes` SQL-string pins; adding POST/PATCH /contacts broke a `test_api_contacts` "read-only → 405" guard (now 422). Targeted pytest passes; CI's full unit+integration fails. Before integrating such a change: run `pytest tests/unit tests/integration` AND grep the touched route/method for `VALUES (%s...` SQL pins and `== 405` read-only guards.
2026-06-11 · audit findings can OVERSTATE gaps: the site-completeness audit listed CSV-import backend, PgCrmStructuredSink, and abuse-controls wiring as missing — all already built (csv_import.py + endpoint + _build_csv_importer; PgCrmStructuredSink; prod_deps passes disposable/velocity/captcha/session_tokens). Before dispatching a build wave off an audit, grep the repo to confirm each "gap" is truly absent — decompose only what's actually missing.
2026-06-11 · new api/*_routes.py modules are wired in BOTH api/app.py (import + ApiDeps field default_factory + `if deps.X is not None: mount_X(...)`) and optionally api/asgi.py (real deps). app.py/asgi.py are boss-only glue — give the builder the new module + tests, boss-wires the mount at integration. Inert-by-default (all-None deps → honest 503) is the house pattern; a DESTRUCTIVE route (account delete) ships inert and requires a deliberate asgi wiring to go live.
2026-06-11 · web/ has TWO parallel UIs: web/src/api/*View.tsx = REAL-MODE authed (ApiClient-backed), web/src/screens/* = MOCK/demo prototype (FLStore). App.tsx renders api/* in the realMode block and screens/* in the mock block; real-mode unhandled routes fall to <ComingSoon>. "Wire feature X to the API" almost always means ADD/EDIT an api/*View component + wire the realMode App.tsx block (boss glue) — NOT touch screens/* (that's the demo, its honesty is separate landing/demo-honesty work). Components take `client?: ApiClient` and fall back to `defaultClient()`.
2026-06-11 · web e2e routes via in-app NAV CLICK, not URL: `route` is useState("dashboard") with NO ?view= routing. To reach a tab in a spec: goto "/", await getByTestId("dashboard-view"), then `.nav-item` hasText:"<Label>".click() (label from src/data.tsx nav arrays). A spec that does goto("/?view=X") silently stays on dashboard and every assertion times out. Real-mode specs run in the chromium-real project (VITE_API_MOCK=0 build) and stub all APIs with page.route — assert the HONEST offline state, never a populated dashboard.
2026-06-11 · pre-wiring a shared web client.ts as boss glue: get the types RIGHT from the backend FIRST (grep the route's return dict / the dataclass.to_dict / asdict fields) — my #228 CsvImportReport/CortexDrift stubs were guesses that were wrong (real: total_rows/imported/rows_upserted/embedded/skipped_unchanged/errors{row,error}; CortexDrift needs registered_auc), so every builder had to "fix" client.ts (territory violation) to compile. Tell builders "client.ts already has your method+types; if a type is wrong REPORT it, don't edit client.ts," and reconcile by patching the boss client.ts against the backend, not by merging N conflicting versions.
